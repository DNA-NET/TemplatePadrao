Grupos

Grupos são as organizações possíveis para os diferentes perfis de usuário. Cada grupo serve para uma ação específica dentro do Atualizado. O administrador do grupo por exemplo tem como função principal administração dos usuários, permissões e gerenciamento de grupos. O Grupo Desenvolvedor é responsável pelo cadastro de seções, templates e regras de funcionamento do site. O Grupo Publicador é responsável pela publicação de conteúdo.

O Worklow que estabelece um fluxo hierárquico para publicação de conteúdo. Somente o Grupo com função de Publicador é capaz de publicar conteúdo no portal, porém é possível definir alguns grupos abaixo do Publicador com função de Autor e Editor. Esses grupos são disponibilizados para atender aos casos em que é preciso estabelecer um fluxo de publicação (Workflow) entre as pessoas responsáveis ​​pela geração de conteúdo, revisão e publicação. Os fluxos começam em um Autor ou Revisor e terminam em um Grupo Publicador que tem privilégios para publicação final no site. 

O Workflow de publicação semper inicia com o grupo de Autor e termina com o Publicador. Entre o Autor e o Publicador poderão ser adicionados vários grupos de Editor conforme a necessidade de cada Workflow. Para inserir um grupo em determinado Workflow, deve-se inicialmente selecionar um grupo superior ao desejado, conforme indica a imagem abaixo.


Permissões

Após a criação de um grupo, devemos definir quais as ações possíveis em cada recurso do ATUALIZA. Para que um Administrador tenha acesso ao cadastro de usuários por exemplo, ele deve ter permissão garantida para acessar este recurso. Cada Grupo pode ter permissões diferentes para ações executadas sobre cada aplicativo do Atualizado. Exemplo:

  • É possível definir se o grupo Publicador pode ter acesso às Seções ou não;
  • É possível definir que tipo de acesso será: apenas consultar ou poderá realizar alterações nas Seções
