Blocos de Conteudo

São Elementos Tipo HTML para utilização no cadastro de conteúdos. A aplicação disponibiliza uma série de elementos prontos para uso, mas o cliente pode personalizar, criando novos elementos para serem utilizados conforme a necessoidade do proejto.

Na listagem de elementos podemos perceber so campos Nome do Elemento, imagem thumb, categoria e Status.

  • Nome do elemento: Pode ser qualquer nome, escolhido pelo Desenvolvedor
  • Thumb: Imagem que será apresentada ao usuário publicador com o padrão visual final do elemento
  • Status: Somente serão disponibilizados para o usuário publicador os elementos ou bocos de conteúdo com status = Ativo

Na tela de cadastro de Blocos de Conteúdo devem ser preenchidos todos os campos.

  • Nome do elemento: Pode ser qualquer nome, escolhido pelo Desenvolvedor
  • Thumb: Imagem que será apresentada ao usuário publicador com o padrão visual final do elemento
  • Categoria: Os blocos podem ser cadastrados em categorias conforme o tipo de elemento. Estas categorias são apresentadas para o usuário publicador como filtro e organização dos elementos.
  • Elemento: Aqui vai o código HTML com o padrão do bloco que deve ser reaproveitado pelo publicador ao selecionar para montar o conteúdo da página.
  • Status: Somente serão disponibilizados para o usuário publicador os elementos ou bocos de conteúdo com status = Ativo

Seções

As seções representam a Organização lógica e hierárquica para classificação dos conteúdos. Contém a arquitetura de informação que define a estrutura completa de navegação do Portal e organização dos repositórios de conteúdo.

A lista de seções exibe toda estrutura de navegação identada, conforme arquitetura de menus definida para o projeto. O sistema admite o cadastro de até 8 níveis de encadeamento de seções.

Entre parênteses ao lado do nome de cada seção consta a quantidade de conteúdo cadastrado para a seção.

Nesta tela incial temos ainda a possibildiade de buscar uma determinada seção para edição ou inclusão de uma nova seção bastando clicar no botão "Incluir Seção" em verde.

Cadastro de Seção

O cadastro de seção requer atenção pela quantidade de configurações possíveis que vão orientar o funcionamento da aplicação.

  • Nome da Seção: Pode ser qualuer nome, conforme planejamento de navegação e arquitera prevista para o portal.
  • Template ou link: Esta é uma das configurações mais importantes. Define o que irá aconecer quando um usuário clica no menu na seção correspondente. Se quisermos por exemplo que ao clicar seja redirecionado para uma página específica, interna ou externa ao portal, devemos colocar o link no campo "Novo Template ou link associado a seção". Enretanto se queremos que abra uma página interna do portal para exibição de conteúdo, devemos selecionar um dos templates disponíveis. Caso o cadastro da seção não tenha previsão de conteúdo associado, para casos onde a seção seve apenas como classificação ou caegorização para navegação, neste caso não devemos informar nenhum Template ou link associado a seção.
  • Url amigável: Cada seção ou cada conteúdo tem uma URL exclusiva para acesso a página. Estas URLs devem ser únicas, não pode ter URL duplicada. A URL é sujerida dinâmicamente com o mesmo nome da seção e sua seção pai, porém o usuário pode modifiacr a URL para uma outra de preferência, desde que não tenha nenhuma outra já cadastrada com o mesmo nome.
  • Posição da seção (paternidade dos menus): Aqui é onde vamos posicionar a nossa nova seção dentro da árvore de seções existentes. Sempre uma seção deve ser filha de uma outra de nível hierárquico maior. Selecione então neste dropdown a seção pai da seção que está sendo criada. Automaticamente após a seleção a URL amigável será atualizada.
  • Imagem conceitual e descrição: É a imagem da seção que pode ser usada opcionalmente na apresentação de uma seção dentro de um portal. Junto com a descrição da seção pode ser o primeiro bloco de conteúdo a ser apresentado no topo de uma página de lista de conteúdo por exemplo. Imagine uma página com lista de notícias e no topo pode aparecer uma imagem maior e um texto introdutório. Para esta finaliidade usamos a imagem conceitual e descrição da seção.
  • Menu interno: Informa se esta seção terá um menu interno para navegação entre os níveis de seções filhas abaixo dela.
  • Abertura: Pode ser em Mesma Janela ou em Nova Janela. Significa que vai abrir a página quando clicar na seção em uma nova ABA ou na mesma ABA do browser.
  • Controle: Este é um controle de acesso ao conteúdo desta seção. Se for Livre, significa que qualuer usuário pode acessar. Se for Restrita Oculta ou Restrita Aparente, somente usuários logados e com direiro de acesso podem acessar o conteúdo desta seção. Na configuração Restrita Aparente, o menu fica aparecendo para o usuário e ao clicar nele verifica se está logado e se tem permissão de acesso, caso contrário redireciona para a página de login. No caso de Rstrita Oculta, a seção só vai aparcer no menu se o usuário já estiver logado e se tiver direito para acesso ao conteúdo da seção.
  • Ordem: Este campo define a ordem de exibição da seção dentro do menu em comparação com as demais seções de mesmo nível. Quanto maior o número da ordem, mais para baixo é posicionada a seção no menu.

Cadastro de Templates

Os templates determinam a formatação de apresentação dos conteúdos que serão exibidos para o usuário de acordo com as configurações e regras programadas que irão fazer o filtro de conteúdo. Os templates são gerados automaticamente no momento em que é executada pela primeira vez a página onde foi criado a partir do nome que o desenvolvedor colocou no código fonte da aplicação.

Exemplo: <?php echo Conteudo("meu_template"); ?>

No exemplo acima, este código irá exibir o conteúdo conforme o padrão definido no template "meu_template". Este template será criado dinâmicamente dentro do sistema após rodar a página em que se encontra pela primeira vez. Após rodar a página, o desenvolvedor deve entar no menu Conteudo>Temlpates, localizar o template novo criado e fazer a configuração desejada.

O novo template gerado dinâmicamente entra no sistema com o tipo incial "Detalhe" que serve para exibir detalhes de um único conteúdo.

Os tipos de template possíveis são:

  • Detalhe: Exibe detalhes de um único conteúdo específico
  • Template: Similar ao Detalhe, porém exibe detalhe de um conteúdo que satisfaça a regra definida no template.
  • Lista de conteúdo: Exibe todos os conteúdos de uma determinada seção que satisfaçam as regrtas de configuração do template.
  • Menu: Exibe somente seções conforme as configurações definidas no template. Uasdo para configuração de menus do site ou listas de seções dentro de página específias

O campo obervações é usado para um detalhamento amigável sobre o comportamento esperado para o template.

  • Conteudo Default: É apresentado o HTML colocado neste campo quando o sistema não consegue localizar nenhum conteúdo que satisfaça as condições de configuração do template
  • Template interno: Aqui vai o HTML padrão a ser consumido pelo sistema para apresentação do conteúdo. Serão substituídos os campos do banco de dados conforme nomenclaturas ididacadas no template interno. Na imagem ao lado por exemplo, está configurado apenas o campo Institucional_texto que traz do banco de dados o corpo do conteúdo na íntegra. Os campos possíveis de serem configurados no template são:
    • Institucional_nome: Título do conteúdo
    • Institucional_url: Campo texto curto, conforme definido na máscara.
    • Institucional_data: Data do conteúdo
    • Institucional_data_cadastro: Data de cadastro do conteúdo
    • Institucional_data_dia: Dia da data do conteúdo
    • Institucional_data_mes: Mes da data do conteúdo
    • Institucional_data_ano: Ano da data do conteúdo
    • Institucional_descricao: Descrição curta do conteúdo conforme definido na máscara
    • Institucional_fonte: Campo texto curto, conforme definido na máscara.
    • Institucional_autor: Campo texto curto, conforme definido na máscara.
    • Institucional_legenda: Campo texto curto, conforme definido na máscara.
    • Institucional_texto: Conteúdo completo.
    • Institucional_imagem_pq: Imagem pequena, thumb do conteúdo
    • Institucional_imagem_gr: Imagem grande do conteúdo
    • Institucional_arquivo: Arquivo associado ao conteúdo
    • Institucional_tamanho: Tamanho do arquivo
    • secao_id: Id da seção
    • conteudo_secao_id: ID da seção principal a qual o conteúdo está associado
    • secao_nome: Nome da seção a qual o conteúdo está associado
    • secao_descricao: Descrição da seção a qual o conteúdo está associado
    • secao_imagem: Imagem da seção a qual o conteúdo está associado
    • secao_link: Link da seção a qual o conteúdo está associado

Web Caching: Quando ativado, o sistema grava o HTML gerado em disco para que nas próximas requisições de conteúdo não seja necessário rodar a regra do template e buscar os dados no banco de dados, desta forma melhorando a performance de carga das páginas que ficam armazenadas em disco. Somente será acessado novamente o banco de dados quando expirar o período do cache que pode ser de 15 minutos, hora em hora, diário, semanal ou mensal.

Regra Geral: Podemos marcar que seja exibido o conteúdo mais atual para o caso de termos mais de um conteúdo cadastrado em uma seção. Também podemos configurar para que seja randômico, ou seja, a cada visualização da página sera exibido um conteúdo diferente sorteado entre os disponíveis. E a última opção serve para o caso de querermos mostrar apenas conteúdos relacionados, ou seja, conteúdos de outras seções que desejamos mostrar no formado de "veja mais", "veja também" ou "matérias relacionadas".

Conteúdo associado as seções: Podemos configurar para trazer somente conteúdos que em como seção principal a seção desejada ou conteúdos que tenham a seção configurada como alternativa. Também é possível selecionar conteúdo que tenha qualquer uma das condições, principal ou alternativa. Abaixo uma imagem ilustrativa da configuração de um conteúdo.

Exibir somente conteúdos com máscara: Exibe somente conteúdo que tem máscara "Página de Conteúdo". Ou seja, se tiver associado a mesma seção outros conteúdos de outras máscaras, somente serão exibidos conteúdos com a máscara "Página de Conteúdo".

Despresar conteudos com máscara (s): Outra forma de configurar o filtro é marcando na coluna da direita quais as máscaras que devem ser desprezadas, desta forma permitindo que seja admitido mais de uma máscara no filtro de conteúdo.

Conteúdo(s) associado(s) a uma ou mais seções: Quando marcamos para usar o mesmo template para a seção corrente. (Secao atual da chamada) estamos informando que o template pode ser reutilizável, buscando conteúdo dentro da seção em qual o usuário está navegando.

Caso seja desejável que busque conteúdo dentro de uma ou mais seções específicas, maracamos então o BOX de cada seção de onde desejamos buscar conteúdo. Aqui no nosso exemplo, como se trata de conteúdo exclusivo para o corpo da home do site, marcamos que desejamos apenas conteúdo cadastrado na seção "home".

Configuração de Template do tipo "Lista de conteúdo"

Para listas de conteúdo valem as mesmas regras que definimos para templates do tipo "Template" com algumas configurações a mais que veremos detalhadamente a seguir.

Em listas de conteúdo é geralmente desejável uma ordenação, ascentende ou descendente dos conteúdos listados seja por ordem alfabética, por data de cadastro ou do coneúdo, ou até outros campos mais específicos. Aqui podemos então configurar até dois níveis de ordenação por até dois campos distintos da lista de conteúdo.

Primeira ordem da lista: Aqui selecionamos qual campo queremos que seja feita a primeira ordenação e também se a ordenação será ascendente ou descendente.

Segunda ordem da lista: Aqui selecionamos qual campo queremos que seja feita a segunda ordenação e também se a ordenação será ascendente ou descendente.

Itens por página: Configuração de quantos conteúdos serão listados na pagina. Ao final da lista na quantidade definida será apresentada a paginação conforme template do site.

Itens por linha: Opcionalmente, podemos configurar uma quantidade de itens por linha, ou seja, dentro da página, podemos dizer que a cada X conteúdos seja exibido um template de quebra conforme definido no campo "Template para quebra de linha" logo abaixo.

Para tempates do tipo menu o que temos de novidade é a possibilidade de configuração de templates de até 5 níveis de menu independentes conforme veremos a seguir.

Cadastro de Máscaras

Determina para o usuário a formatação da inclusão para cada tipo de conteúdo. Conteúdos podem ter naturezas diferentes que necessitem de uma configuração personalizada com campos e definições próprias. Por exemplo, se o nosso coneúdo for uma notícia, podemos ter vários campos definidos para qualificar adequadamente o conteúdo. Podemos ter além do título da notícia, data e da noticia em si, uma série outros campos, como linha fina, cartola, autor, foto de destaque, legenda da foto, descrição resumida, categoria, etc. 

Para facilitar a entrada de dados para os usuários publicadores, criamos então as máscaras com os campos e formatações específicas conforme o caso de uso planejado para o projeto do nosso portal.

Na imagem mostramos alguns casos de usos de máscaras como áudios, banners, certificados, depoimentos, galeria de imagens, licitações, notícias, perguntas frequentes, etc. onde para cada caso teremos campos e formatações diferentes.

Em uma máscara de "Perguntas frequentes" por exemplo, nos interessa ter dois campos apenas, a pergunta e a resposta. Já a máscara de notícias ou licitações vão ter muitos outros campos pra o usuário publicador preencher.

Ao configurar uma máscara, vamos definir os campos, e regras conforme demostrarei a seguir utilizando a máscara de "Notícias" como exemplo.

Nome da máscara: Aqui definimos o nome da máscara. No exemplo, o nome é "Notícias".

Texto de ajuda para a Mascara: Aqui vai um texto explicativo para o usuário compreender qual tipo de conteúdo e para que serve.

Campo título ou nome: Este é um campo obrigatório, corresponde ao título do conteúdo contextualizado para a situção da máscara. Utilizando como notícias, no exemplo definimos como "Titulo da Notícia" e no campo abaixo marcamos como de preenchimento obrigatório. Podemos também marcar uma formatação específica para o tipo de campo:

Texto de ajuda para o campo: Aqui colocamos instruções de preenchimento que serão exibidas como tolltip de ajuda para o usuário, caso tenha dúvudas na hora do preenchimento do campo.

Valores fixos para este campo: (Combo box): Aqui utilizamos quando desejamos que seja apresentado ao usuário valores fixos para seleção, utilizado por exemplo para o caso de categorias de notícias.

Campo Texto: Aqui vai o texto completo do conteúdo, no caso de uma notícia seria a notícias na ínegra com todos os parágrafos, imagens, etc. Podemos para o preenchimentode campos texto optar em liberar para o usuário uma caixa de texto simples, um pequeno editor HTML ou um editor sofisticado que permite a edição a partir de blocos de conteudo ou elementos pre-formatados e custimizados conforme linha criativa do projeto.

Imagens: Para cada máscara é possível definir dois campos de imagens de destaque, geralmente posicionadas na home ou nas listas de conteúdo como imagem padrão ou desaque do conteúdo.

Também conforme o projeto de design do site, podemos definir os tamanhos da imagem corretos na máscara para facilitar ao usuário o cadastro das imagens sem se preocupar com o tamanho da imagem original. O sistema na hora do cadastro da imagem vai apresentar ao usuário uma ferramenta de edição que permite fazer o corte ou crop da imagem conforle a largura e altura definidos na máscara de conteúdo.

Arquivos: Para cada máscara é possível tamém a utilização de dois campos de arquivos. Estes campos são utilizados por exemplo quando temos a necessidade de cadastrar documentos em uma lista de documentos. Ou algum anexo obrigatório como uma imagem de alta resolução ou vídeo para Download.

Ao configurar a máscara, o desenvolvedor pode definir uma pasta de destino para os arquivos específicos desta máscara. As pastas serão criadas dentro da pasta "conteudo" e estarão acessíveis ao publicador por dentro da midiateca do site.

Além da pasta é possível especificar as extensões de arquivos permitidos e tamanho limite em Mb para o upload de arquivos.


Seção preferencial: Aqui podemos pre-definir a seção onde será cadastrado conteúdo com a máscara que está sendo configurada. No nosso exemplo, cuja máscara é de "Notícias" selecionamos como seção preferencial a seção Home>imorensa>notícias. Ou seja, quando o usuário for incluir uma nova notícia ao selecionar a máscara, já vem marcada a seção notícias e o usuário não precisa marcar a seção.
